# Test Generation Architecture Specification

## Overview

This document specifies the exact requirements for the new test generation and execution system for IaRa.

## Step 1: Regenerate the Tests

**What:** Process `experiments.yaml` files to generate one `CMakeLists.txt` **per experiment** (not per instance) that encodes **all experiment instances** for that experiment set.

**Input:**
- All `applications/*/experiment/experiments.yaml` files
- For each file, identify experiment sets marked with `generate_tests: true`

**Output:**
- Generate `applications/<XX-experiment-name>/experiment/CMakeLists.txt`
- This single CMakeLists.txt file contains all instances for all `generate_tests: true` experiment sets in that application

**Script:** `generate_experiments.py` (to be created in `scripts/`)

**Instance Naming Convention:**
```
<XX-experiment-name>_<scheduler>_<parameter-name-1>_<parameter-value-1>_<parameter-name-2>_<parameter-value-2>_...
```

Examples:
- `05-cholesky_vf-omp_num-blocks_32_matrix-size_2048`
- `05-cholesky_sequential_matrix-size_2048_num-blocks_4`
- `03-two-nodes_vf-sequential`
- `06-broadcast_vf-omp`

**Key Requirements:**
- No separate directory per test instance
- Parameter names and values are both part of the instance name
- Parameters appear in the order they are defined in the YAML
- The scheduler parameter is handled specially and appears right after the experiment name
- Single CMakeLists.txt file per application, not per instance

## Step 2: Reconfigure CMake

**What:** Run CMake with the generated CMakeLists.txt files to create build directories and define test targets.

**Build Directory Structure:**
```
iara/build_experiments/<instance_name>/
```

Where each instance gets its own directory directly under `build_experiments/`.

Example:
```
build_experiments/
├── 05-cholesky_vf-omp_num-blocks_32_matrix-size_2048/
├── 05-cholesky_sequential_matrix-size_2048_num-blocks_4/
├── 03-two-nodes_vf-sequential/
├── 06-broadcast_vf-omp/
└── ...
```

**Generated Test Targets:**

For each instance, CMake should define exactly 2 test targets:

1. **build-<instancename>**
   - Compiles the instance
   - Executes the full pipeline:
     - Generate topology (if necessary)
     - Call iara-opt with the correct scheduler
     - Lower schedule with mlir-to-llvmir
     - Compile to executable
   - All test particularities/parameters must be preserved from the old framework
   - Success = executable is created
   - Failure = any step fails (non-zero exit code)

2. **run-<instancename>**
   - Executes the compiled executable
   - Depends on: `build-<instancename>`
   - Success = executable returns 0
   - Failure = executable returns non-zero or times out

**CMake Configuration Command:**
```bash
cmake -B iara/build_experiments \
    -DCMAKE_BUILD_TYPE=Release \
    -DIARA_BUILD_TESTS=ON \
    .
```

## Step 3: Run the Tests

**What:** Execute all generated tests via CTest.

**Dependency Model:**
- Each `run-<instancename>` test depends on its matching `build-<instancename>` test
- CMake (via CTest) respects these dependencies
- A failed build-* test causes the corresponding run-* test to not execute
- A failure to build is represented by a failed test

**Execution:**
```bash
ctest --build-dir build_experiments --build-config Release -V
```

## Integration with build_tests.sh

The `scripts/build_tests.sh` script orchestrates all three steps:

```bash
#!/bin/bash

# Step 1: Regenerate tests
python3 scripts/generate_experiments.py

# Step 2: Reconfigure CMake
cmake -B build_experiments \
    -DCMAKE_BUILD_TYPE=Release \
    -DIARA_BUILD_TESTS=ON \
    .

# Step 3: Run tests
ctest --build-dir build_experiments --build-config Release -V
```

## Example Walkthrough

### Input: applications/05-cholesky/experiment/experiments.yaml

```yaml
application:
  name: cholesky
  entry: "05-cholesky"
  sources:
    app_dir: "applications/cholesky/src"
  build:
    extra_linker_args: ["-lopenblas"]
    defines: ["VERBOSE"]

parameters:
  - name: matrix_size
    type: int
  - name: num_blocks
    type: int
  - name: scheduler
    type: str
    choices: [sequential, vf-omp]

constraints:
  - expression: "matrix_size % num_blocks == 0"

experiment_sets:
  - name: regression_test
    parameters:
      matrix_size: [2048]
      num_blocks: [2, 4]
      scheduler: [sequential, vf-omp]
    generate_tests: true
```

## Key Technical Constraints

1. **One CMakeLists.txt per application**, not per instance
2. **No deep directory nesting** - all instances build to `build_experiments/<instance_name>/`
3. **Parameter order matters** - must preserve order from YAML definition
4. **Scheduler is special** - appears immediately after experiment name in instance name
5. **Full pipeline preservation** - must use exact same compilation steps as old framework
6. **Test particularities** - all build/defines/linker_args from YAML must be passed through
7. **Dependency enforcement** - run-* must depend on build-*, CTest must respect this

## Success Criteria

1. `generate_experiments.py` parses all `experiments.yaml` files correctly
2. CMakeLists.txt files are generated with correct instance names and all parameters
3. CMake configuration creates build directories with the flat naming scheme
4. All `build-*` and `run-*` targets are created as CTest tests
5. Dependencies are properly set up (run-* depends on build-*)
6. `build_tests.sh` orchestrates all three steps correctly
7. All tests can be executed via `ctest`
