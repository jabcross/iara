## Degridder (13-degridder) — SPRINT TARGET
- Radio astronomy degridding: sky image → visibilities
- Adapted from DDFacet (originally Python/NumPy) via Preesm piSDF implementation by Anaëlle Cloarec

### Data Flow
1. `generate_kernel` → degridding kernels (float2 array) + kernel supports (int2 array)
2. `config_struct_set_up` → Config struct with file paths, frequency, scales
3. `load_image_from_file` → reads CSV image grid (GRID_SIZE x GRID_SIZE floats)
4. `visibility_host_set_up2` → reads visibility UVW coords from .csv or .ms (MeasurementSet via casacore)
5. `fft_shift_real_to_complex_actor` → FFT shift
6. `CUFFT_EXECUTE_FORWARD_C2C_actor` → forward FFT (using FFTW, not CUDA despite name)
7. `fft_shift_complex_to_complex_actor` → another FFT shift
8. `degridder_actor` (sub-actor): broadcasts inputs to NUM_CHUNK copies, runs `std_degridding_parallel2` on each chunk, gathers results
9. `convert_vis_to_csv` → writes output

### Input Data
- **Image grid**: CSV file with GRID_SIZE*GRID_SIZE floats (one per line)
  - `fits_to_csv.py` converts FITS→CSV (Gemini-generated, potential segfault source)
  - Expected path: `data/input/cycle_0_deconvolved_{2560|5120}.csv`
- **Visibilities**: Either .csv (6 floats per line: u v w real imag weight) or .ms (MeasurementSet, via casacore C++ wrapper)
  - CSV format: first line = num_visibilities count, then `%lf %lf %lf %lf %lf %lf` per line
  - MS format: loads via casacore columns (UVW, DATA, WEIGHT)
  - Expected path: `data/sim_{small|medium|large}.ms`
- **Kernels**: CSV files for real, imaginary, and support values
  - Expected path: `data/input/kernels/small/w-proj_*.csv`
- `onetime.sh`: Downloads all data from Google Drive via gdown, unzips .ms files, runs fits_to_csv.py

### Key Observations for Phase 3 (Bugfixing)
- **The segfault is with regenerated CSV inputs**. The `load_image_from_file` function reads CSV with `fscanf(f, "%f%*[, \n]", &output[i])` — parses floats separated by comma, space, or newline.
- `fits_to_csv.py` writes one float per line with `np.savetxt(..., fmt='%f')` — should be compatible.
- `visibility_host_set_up2` now supports .ms (MeasurementSet) directly via casacore, bypassing CSV for visibility data.
- The `Config` struct is populated by `config_struct_set_up` which hardcodes file paths relative to `DATA_ROOT_PATH` (a generated define).
- Constants are compile-time defines: GRID_SIZE, NUM_VISIBILITIES, NUM_KERNELS, etc.
- The topology template uses placeholder substitution (GRID_SIZE_VALUE → actual value).

### Potential Segfault Causes (to investigate in Phase 3)
- CSV format mismatch in `load_image_from_file`: wrong number of values, wrong delimiter, or NaN/Inf
- Buffer size mismatch: GRID_SIZE*GRID_SIZE vs actual CSV element count
- The `fits_to_csv.py` script may produce wrong dimensions (FITS image may not be GRID_SIZE x GRID_SIZE)
- The FITS image may have different axes ordering or extra dimensions
- The Config struct passed through the dataflow may have padding/alignment issues between C and LLVM struct layout
