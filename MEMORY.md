## 2. Architectural Invariants
*   **Virtual FIFO (VF) Runtime:** Current standard for degridder; uses keyed semaphores (`gtl::parallel_flat_hash_map`) for synchronization.
*   **One Pass, Multiple Tasks:** The `VirtualFIFOSchedulerPass` is monolithic by design to avoid expensive IR-triggered recompilations and to maintain shared state (hash maps) during lowering.
*   **Macro-Augmented IR:** Operations use `CREATE` and `DEF_OP` macros to embed compiler source-code locations for easier debugging and better IDE/LSP support.
*   **Build Isolation:** The experiment framework must use one CMake invocation per instance (`IARA_EXPERIMENT_INSTANCE`) to ensure clean builds.
*   **Parameter Mapping:** (Standardizing) Parameters move from hardcoded Python logic to `computed_parameters` in application-specific `experiments.yaml` files.
*   **Environment Management:** Spack is required for RISC-V target support; `.env.cached` is used to bypass slow Spack activation cycles.

## 3. Dependency Graph
### **Compiler (`iara-opt`)**
*   **Dialect:** `ActorOp` (containers), `NodeOp` (impl references), `EdgeOp` (multi-rate/delay connections).
*   **Passes:** `Flatten` → `Canonicalize` → `VirtualFIFOScheduler` (SDF analysis, Buffer sizing via Presburger, Codegen).

### **Runtime (`runtime/`)**
*   **Virtual FIFO:** `Chunk` (memory slices), `Edge` (push/propagate), `Node` (consume/fire).
*   **Work-Stealing:** Defaulting to OpenMP or EnkiTS.
*   **Semaphores:** Keyed semaphores manage firing dependencies based on `arg_bytes`.

### **Tools (`tools/experiment_framework`)**
*   **Generator:** Creates `CMakeLists.txt` from `experiments.yaml`.
*   **Builder:** Clean → Config → Build → Measure.
*   **Collector:** Parses execution time and statistics.
