## Executive Summary

### Project Goal

Consolidate 20+ scattered Python scripts from `applications/experiment-framework/` into a unified CLI tool that provides declarative, parameterized experiment generation, execution, and visualization for the IaRa compiler project.

### Key Requirements

1. **Single Source of Truth**: `experiments.yaml` drives all configuration
2. **Per-Set Isolation**: Each experiment set gets isolated build directory with forced clean builds
3. **Sequential Execution**: All instances in a single batch, no build concurrency
4. **Composite Timing**: Measure compilation via GNU time for `iara-opt`, `mlir-to-llvm`, and `clang`
5. **Base SI Units**: Store all measurements in base units (bytes, seconds), convert for display
6. **Error Recovery**: Retry failed instances twice, continue on failure, report in notebooks
7. **Vega-Altair Visualization**: Full Vega-Lite JSON specs in YAML
8. **Hash-Based Change Detection**: SHA256 hash of YAML with hard failure on mismatch

### Deferred Features

- **Slurm Integration**: Documented in `TODO.md`, not implemented in Phase 1

### Component Interaction Flow

```
experiments.yaml
    ↓
[config.py] Parse & Hash → .experiments.yaml.hash
    ↓
[generator.py] Generate → CMakeLists.txt
    ↓
[CMake] Configure → build/<set>/<instance>/
    ↓
[builder.py] Sequential Build + GNU time
    ↓ (writes)
results_<timestamp>.json (compilation + binary data)
    ↓
[collector.py] Execute instances + append measurements
    ↓ (appends)

    ↓
[visualizer.py] Generate Vega-Lite specs
    ↓
[notebook.py] Create Jupyter notebook with plots + failure report
    ↓
analysis_<timestamp>.ipynb
```

## Component Specifications

### 1. CLI Interface (`cli.py`)

**Purpose**: Unified command-line interface for all operations

**Commands**:

#### `generate`
```bash
python3 -m tools.experiment_framework generate \
    --app <XX-app-name> \
    --set <experiment-set-name>
```

**Behavior**:
- One-time operation (fails if hash exists and differs)
- Computes SHA256 hash of `experiments.yaml`
- Generates `CMakeLists.txt` with `iara_add_test_instance()` calls
- Writes `.experiments.yaml.hash`
- **Exit Codes**: 0=success, 1=hash mismatch, 2=invalid YAML

#### `build`
```bash
python3 -m tools.experiment_framework build \
    --app <XX-app-name> \
    --set <experiment-set-name> \
    [--clean]  # Default: true (forced clean build)
```

**Behavior**:
- Forces clean build of entire experiment set
- Sequentially builds all instances (no concurrency)
- Wraps compilation steps with GNU time
- Retries failed instances (max 2 attempts)
- Writes `results_<timestamp>.json` with compilation + binary data
- **Exit Codes**: 0=all success, 1=some failures (but continued), 2=critical error

#### `execute`
```bash
python3 -m tools.experiment_framework execute \
    --app <XX-app-name> \
    --set <experiment-set-name> \
    --repetitions <N>  # Default: from YAML execution.repetitions
    --timeout <seconds>  # Default: from YAML execution.timeout
```

**Behavior**:
- Executes all successfully built instances
- Runs each instance N times (repetitions)
- Appends runtime measurements to existing JSON
- Skips failed build instances
- **Exit Codes**: 0=success, 1=some execution failures, 2=no instances to run

#### `visualize`
```bash
python3 -m tools.experiment_framework visualize \
    --app <XX-app-name> \
    --set <experiment-set-name> \
    --results <path-to-json>  # Default: latest in results/
```

**Behavior**:
- Reads Vega-Lite specs from `experiments.yaml`
- Generates standalone `.vl.json` files with data URLs
- Creates Jupyter notebook with embedded visualizations + failure report
- Executes notebook via `nbconvert --execute --inplace`
- **Exit Codes**: 0=success, 1=visualization errors (partial output), 2=no data

#### `run` (Orchestrator)
```bash
python3 -m tools.experiment_framework run \
    --app <XX-app-name> \
    --set <experiment-set-name> \
    [--skip-generate]  # Skip generation if CMakeLists.txt exists
    [--skip-build]     # Use existing binaries
    [--skip-execute]   # Use existing measurements
```

**Behavior**:
- Orchestrates full pipeline: generate → build → execute → visualize
- Stops on critical errors (invalid YAML, hash mismatch)
- Continues on instance failures (reports in notebook)
- **Exit Codes**: 0=full success, 1=partial success, 2=pipeline failure

**Global Options** (all commands):
```bash
--verbose, -v       # Increase output verbosity
--quiet, -q         # Suppress non-error output
--log <file>        # Write detailed log to file
```

```python
    Check if YAML content has changed since last generation.
    
    Returns:
        True if changed or hash file doesn't exist
        False if unchanged
    """

def store_yaml_hash(yaml_path: Path, hash_file: Path) -> None:
    """Write computed hash to .experiments.yaml.hash file."""

def get_parameter_combinations(
    config: Dict[str, Any],
    experiment_set: str
) -> List[Dict[str, Any]]:
    """
    Generate all valid parameter combinations for an experiment set.
    
    Process:
        1. Extract parameter definitions from config
        2. Get parameter values from specified experiment set
        3. Compute Cartesian product of all parameter values
        4. Apply constraints to filter invalid combinations
        5. Compute derived parameters for each combination
    
    Returns:
        List of parameter dictionaries, each representing one instance
    """
```

**YAML Schema Validation**:

Required top-level keys:
- `application` (dict): Application metadata
- `parameters` (list): Parameter definitions
- `measurements` (list): Measurement specifications
- `experiment_sets` (list): Experiment set definitions

Optional top-level keys:
- `computed_parameters` (list): Derived parameter expressions
- `constraints` (list): Constraint expressions
- `execution` (dict): Execution configuration
- `visualization` (dict): Visualization specifications

**Purpose**: Generate CMakeLists.txt from experiments.yaml

```python
def generate_instance_name(
    app_name: str,
    params: Dict[str, Any]
) -> str:
    """
    Generate instance name following naming convention.
    
    Format:
        <XX-app-name>_<scheduler-value>_<param-name-1>_<value-1>_...
    
    Rules:
        - Application name must start with two digits
        - Scheduler value only (no "scheduler=" prefix)
        - All other parameters include name and value
        - Parameters separated by underscores
        - Values sanitized (spaces → hyphens, special chars removed)
    
    Example:
        app_name = "05-cholesky"
        params = {"scheduler": "vf-omp", "matrix_size": 2048, "num_blocks": 4}
        → "05-cholesky_vf-omp_matrix-size_2048_num-blocks_4"
    """

            NAME "05-cholesky_vf-omp_matrix-size_2048_num-blocks_2"
            EXPERIMENT_SET "regression_test"

            ENTRY "05-cholesky"

            BUILD_DIR "${CMAKE_BINARY_DIR}/05-cholesky/regression_test/..."
            DEFINES "MATRIX_SIZE=2048" "NUM_BLOCKS=2"
```

### 4. Builder Module (`builder.py`)

**Purpose**: Execute builds with timing measurement and retry logic

**Key Functions**:

```python
def build_instance(
    instance_name: str,
    build_dir: Path,
    cmake_source: Path,
    attempt: int = 1
) -> BuildResult:
    """
    Build a single instance with timing measurement.
    
    Process:
        1. Force clean: rm -rf build_dir/*
        2. Configure: cmake -B build_dir -S cmake_source
        3. Build with timing wrappers:
           a. /usr/bin/time -v -o iara_opt_time.txt iara-opt ...
           b. /usr/bin/time -v -o mlir_llvm_time.txt mlir-to-llvm ...
           c. /usr/bin/time -v -o clang_time.txt clang ...
        4. Parse timing output with regex
        5. Extract binary size: size -A <executable>
        6. Convert all units to base SI
    
    Returns:
        BuildResult with:
            - success: bool
            - instance_name: str
            - compilation: dict (iara_opt_time_s, mlir_llvm_time_s, clang_time_s,
                                 iara_opt_memory_bytes, ...)
            - binary_size_bytes: int
            - errors: List[str] (if failed)
            - attempt: int
    """

def parse_time_output(time_file: Path) -> Dict[str, Any]:
    """
    Parse GNU time -v output with regex.
    
    Patterns:
        User time (seconds): (\d+\.?\d*)
        System time (seconds): (\d+\.?\d*)
        Elapsed (wall clock) time: (\d+):(\d+\.?\d*)  # OR (\d+\.?\d*)
        Maximum resident set size (kbytes): (\d+)
        Minor (reclaiming a frame) page faults: (\d+)
        Major (requiring I/O) page faults: (\d+)
    
    Unit Conversions:
        - Time: always seconds (float)
        - Memory: kbytes → bytes (* 1024, int)
    
    Error Handling:
        - Missing fields: return None for that field, log warning
        - Invalid format: raise ParseError with line context
    
    Returns:
        Dict with keys: user_time_s, system_time_s, wall_time_s,
                       max_rss_bytes, minor_faults, major_faults
    """

def get_binary_size(executable: Path) -> int:
    """
    Extract binary size using 'size -A'.
    
    Returns:
        Total size in bytes (sum of .text, .data, .bss, .rodata sections)
    """

def build_all_instances(
    instances: List[str],
    base_build_dir: Path,
    cmake_source: Path,
    max_retries: int = 2
) -> BuildResults:
    """
    Build all instances sequentially with retry logic.
    
    Process:
        For each instance:
            attempt = 1
            while attempt <= max_retries:
                result = build_instance(instance, ..., attempt)
                if result.success:
                    break
                attempt += 1
            
            if not result.success:
                track as failed, continue to next instance
    
    Returns:
        BuildResults with:
            - successful_instances: List[BuildResult]
            - failed_instances: List[FailedBuildResult]
              (includes all error messages from all attempts)
            - timestamp: str (ISO 8601)
    """
```

**Timing Output Files** (per instance):

```
build/<set>/<instance>/
├── iara_opt_time.txt        # GNU time output for iara-opt
├── mlir_llvm_time.txt       # GNU time output for mlir-to-llvm
├── clang_time.txt           # GNU time output for clang
├── build.log                # Full build stdout/stderr
└── <executable>             # Final binary
```

### 5. Collector Module (`collector.py`)

**Purpose**: Execute instances and collect runtime measurements

**Key Functions**:

```python
def execute_instance(
    executable: Path,
    repetitions: int,
    timeout: int,
    env_vars: Dict[str, str],
    measurements: List[MeasurementSpec]
) -> ExecutionResult:
    """
    Execute instance with GNU time and extract measurements.
    
    Process:
        For i in range(repetitions):
            1. Run: /usr/bin/time -v <executable>
            2. Capture stdout and time stderr
            3. Parse measurements using regex patterns from YAML
            4. Convert to base SI units
            5. Store run data
    
    Returns:
        ExecutionResult with:
            - instance_name: str
            - runs: List[RunResult]  # One per repetition
            - failures: List[str]    # Errors from failed runs
    """

def parse_measurement(
    output: str,
    spec: MeasurementSpec
) -> Optional[Union[float, int, str]]:
    """
    Extract single measurement using parser from YAML.
    
    Parser Types:
        - regex: Apply pattern, extract group, convert to type
        - json: Parse JSON, extract key path
        - line: Extract specific line number
    
    Unit Conversion:
        Apply conversion from spec unit to base SI:
        - ms → s (/ 1000.0)
        - KB → bytes (* 1024)
        - MB → bytes (* 1024 * 1024)
        - GB → bytes (* 1024 * 1024 * 1024)
    
    Returns:
        Converted value or None if not found/required=false
    
    Raises:
        MeasurementError: If required measurement missing
    """

def collect_all_measurements(
    build_results: BuildResults,
    config: Dict[str, Any],
    repetitions: int,
    timeout: int
) -> CollectionResults:
    """
    Execute all successful instances and collect measurements.
    
    Process:
        For each successful instance:
            1. Get executable path from build result
            2. Extract environment variables from config
            3. Execute with specified repetitions
            4. Collect all measurements from runs
            5. Track execution failures
    
    Returns:
        CollectionResults with:
            - execution_results: List[ExecutionResult]
            - timestamp: str
    """
```

```python
def convert_units_for_display(
    data: pd.DataFrame,
    measurement: str,
    base_unit: str
) -> Tuple[pd.DataFrame, str, float]:
    """
    Convert from base SI to appropriate display unit.
    
    Conversion Strategy:
        Memory (bytes):
            < 1024        → bytes (1)
            < 1024²       → KB (1024)
            < 1024³       → MB (1024²)
            >= 1024³      → GB (1024³)

            < 0.001       → μs (1e-6)
            < 1.0         → ms (0.001)
            >= 1.0        → s (1)
    
    Returns:
        (converted_data, display_unit, scale_factor)
    
    Example:
        data['value'] = [1048576, 2097152]  # bytes
        → ([1.0, 2.0], "MB", 1048576)
```

**New Function**: `iara_add_test_instance()`

```cmake
function(iara_add_test_instance)
    cmake_parse_arguments(
        INSTANCE
        ""  # No boolean flags
        "NAME;EXPERIMENT_SET;APPLICATION_DIR;ENTRY;SCHEDULER;BUILD_DIR"
        "PARAMETERS;DEFINES;LINKER_ARGS;COMPUTED_PARAMS"
        ${ARGN}
    )
    
    # Extract parameters into defines
    set(ALL_DEFINES ${INSTANCE_DEFINES})
    foreach(PARAM ${INSTANCE_PARAMETERS})
        string(REPLACE "=" ";" PARAM_PAIR ${PARAM})
        list(GET PARAM_PAIR 0 PARAM_NAME)
        list(GET PARAM_PAIR 1 PARAM_VALUE)
        
        # Convert to uppercase for define
        string(TOUPPER ${PARAM_NAME} PARAM_UPPER)
        list(APPEND ALL_DEFINES "${PARAM_UPPER}=${PARAM_VALUE}")
    endforeach()
    
    # Set build directory for this instance
    set(INSTANCE_BUILD_DIR "${INSTANCE_BUILD_DIR}")
    
    # Call existing iara_add_application with per-instance configuration
    iara_add_application(
        SCHEDULER ${INSTANCE_SCHEDULER}
        TARGET_NAME "${INSTANCE_NAME}"
        BUILD_DIR "${INSTANCE_BUILD_DIR}"
        APP_SRC_DIR "${INSTANCE_APPLICATION_DIR}/src"
        ENTRY "${INSTANCE_ENTRY}"
        DEFINES ${ALL_DEFINES}
        EXTRA_LINKER_ARGS ${INSTANCE_LINKER_ARGS}
    )
    
    # Create test targets
    add_custom_target(
        "build-${INSTANCE_NAME}"
        DEPENDS "${INSTANCE_NAME}"
    )
    
    add_test(
        NAME "run-${INSTANCE_NAME}"
        COMMAND "${INSTANCE_BUILD_DIR}/${INSTANCE_NAME}"
        WORKING_DIRECTORY "${INSTANCE_BUILD_DIR}"
    )
    
    set_tests_properties(
        "run-${INSTANCE_NAME}"
        PROPERTIES
            DEPENDS "build-${INSTANCE_NAME}"
            TIMEOUT ${INSTANCE_TIMEOUT}
    )
endfunction()
```

**Modifications to `iara_add_application()`**:
- Accept `BUILD_DIR` parameter (override default)
- Support per-instance isolated builds
- Preserve all existing functionality

### Vega-Lite Integration

**Display Unit Selection Algorithm**:

```python
def select_display_unit(values: List[float], base_unit: str) -> Tuple[str, float]:
    """
    Auto-select appropriate display unit based on magnitude.
    
    Args:
        values: List of values in base SI units
        base_unit: Either "bytes" or "seconds"
    
    Returns:
        (display_unit, scale_factor)
    """
    if base_unit == "bytes":
        max_value = max(values)
        if max_value >= 1024**3:
            return ("GB", 1024**3)
        elif max_value >= 1024**2:
            return ("MB", 1024**2)
        elif max_value >= 1024:
            return ("KB", 1024)
        else:
            return ("bytes", 1)
    
    elif base_unit == "seconds":
        max_value = max(values)
        if max_value >= 1.0:
            return ("s", 1.0)
        elif max_value >= 0.001:
            return ("ms", 0.001)
        else:
            return ("μs", 0.000001)
    
    return (base_unit, 1.0)
```

## Error Handling and Recovery

### Retry Logic

**Strategy**: Immediate retry on failure

```python
def build_with_retry(
    instance: str,
    max_attempts: int = 2
) -> BuildResult:
    """
    Build instance with immediate retry on failure.
    
    Process:
        attempt = 1
        errors = []
        
        while attempt <= max_attempts:
            result = build_instance(instance, attempt)
            
            if result.success:
                return result
            
            errors.append({
                "attempt": attempt,
                "error": result.error,
                "timestamp": datetime.now().isoformat()
            })
            
            log.warning(f"Attempt {attempt} failed: {result.error}")
            attempt += 1
        
        # All attempts failed
        return FailedBuildResult(
            instance=instance,
            attempts=max_attempts,
            errors=errors
        )
    """
```

### A. Naming Conventions

**Instance Names**:
- Format: `<XX-app>_<scheduler>_<param1-name>_<value1>_<param2-name>_<value2>_...`
- Scheduler: Value only (no "scheduler=" prefix)
- Other params: Include name and value
- Separators: Underscores
- Value sanitization: Replace spaces with hyphens, remove special characters

**File Names**:
- Results: `results_YYYYMMDD_HHMMSS.json`
- Plots: `plot_<name>_YYYYMMDD_HHMMSS.vl.json`
- Notebooks: `analysis_YYYYMMDD_HHMMSS.ipynb`
- Timing output: `<instance>_<tool>_time.txt`

### B. Regex Patterns

**GNU Time Parsing**:
```python
PATTERNS = {
    "user_time": r"User time \(seconds\): (\d+\.?\d*)",
    "system_time": r"System time \(seconds\): (\d+\.?\d*)",
    "wall_time": r"Elapsed \(wall clock\) time \(h:mm:ss or m:ss\): (?:(\d+):)?(?:(\d+):)?(\d+\.?\d*)",
    "max_rss": r"Maximum resident set size \(kbytes\): (\d+)",
    "minor_faults": r"Minor \(reclaiming a frame\) page faults: (\d+)",
    "major_faults": r"Major \(requiring I/O\) page faults: (\d+)",
}
```

**Wall Time Format Handling**:
- `ss.ss` → seconds only
- `mm:ss.ss` → minutes * 60 + seconds
- `h:mm:ss.ss` → hours * 3600 + minutes * 60 + seconds
